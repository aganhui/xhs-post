# xhs-copy: 小红书风格文案生成

cc 直接用当前 LLM 生成小红书风格文案，**严格遵守以下规则**。免费，用自己的算力。

## 输出格式

```
标题：<20权重的标题>
正文：<200-500字正文>

#标签1 #标签2 #标签3
```

## 标题规则

- **权重计算**：中文字符=1权重，英文/数字=0.5权重，总权重≤20
- **要吸引人**：说结果/说感受/制造好奇，允许1个emoji开头
- 好标题："这3个AI工具让我准时下班"、"Obsidian这招太香了"、"后悔没早用的笔记法"
- 烂标题："推荐三个工具"、"我的知识管理方法"、"用obsidian和claude"

### 标题权重计算参考

```
"这3个AI工具让我准时下班"
这(1)+3(0.5)+个(1)+A(0.5)+I(0.5)+(0.5)+工(1)+具(1)+让(1)+我(1)+准(1)+时(1)+下(1)+班(1) = 12 ✅

"后悔没早用的笔记法"
后(1)+悔(1)+没(1)+早(1)+用(1)+的(1)+笔(1)+记(1)+法(1) = 9 ✅
```

## 正文规则

- 200-500字，说话自然，像跟同事安利好东西
- 不要排比句、不要每段加emoji、不要感叹号满天飞
- 标签3-5个，用 `#标签` 格式写在正文最后，空一行隔开

## 禁止套路词

谁懂啊、家人们、亲测、码住、建议收藏、绝绝子、拿走不谢、冲冲冲、狠狠

## 文案风格参考

不同主题有不同写法倾向：

| 主题类型 | 标题倾向 | 正文倾向 | 示例 |
|---------|---------|---------|------|
| 好物推荐 | 说效果/对比 | 逐一介绍+个人体验 | "这3个AI工具让我准时下班" |
| 方法分享 | 说结果/逆袭 | 步骤清晰+避坑提醒 | "Obsidian这招太香了" |
| 知识科普 | 制造好奇 | 通俗解释+举例 | "90%人不知道的搜索技巧" |
| 生活记录 | 说感受/共鸣 | 真实细节+情绪 | "一个人住后的10个变化" |

## 使用方式

在 cc 中说：
- "帮我写一篇小红书文案，主题是XXX" → 直接生成
- "帮我发一篇小红书，主题是XXX" → /xhs-post 主流程自动调用
